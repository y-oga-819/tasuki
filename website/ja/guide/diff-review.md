# Diffレビュー

TasukiのDiffビューアは`@pierre/diffs`を基盤としており、Shadow DOM内でレンダリングすることで完全なスタイル分離を実現しています。

## ビューモード

### Split View

左に旧ファイル、右に新ファイルを並べて表示するサイドバイサイド比較。デフォルトかつ最も直感的なレビュー方法です。

### Unified View

削除行と追加行をインラインで表示するシングルカラムビュー。小さな変更や画面幅が限られている場合に便利です。

ツールバーの **Split / Unified** ボタンで切り替えます。

## ツールバーコントロール

| コントロール | 説明 |
|------------|------|
| **Split / Unified** | Diffレイアウトの切替 |
| **Scroll / Wrap** | 長い行の折り返し/横スクロール切替 |
| **Expand / Collapse** | 未変更コンテキスト行の表示/非表示 |
| **Stats** | 変更ファイル数、追加行数、削除行数を表示 |

## ファイルナビゲーション

サイドバーにはステータスインジケータ付きの変更ファイル一覧が表示されます：

| アイコン | ステータス |
|---------|-----------|
| **M**（青） | 変更 |
| **A**（緑） | 追加 |
| **D**（赤） | 削除 |
| **R**（紫） | リネーム |

ファイル名をクリックすると該当のdiffまで直接スクロールします。ファイルヘッダー（ファイル名が表示されているバー）をクリックすると、そのファイルのdiffを折りたたみ/展開できます。

## 生成ファイル

`is_generated`フラグが付いたファイル（`package-lock.json`など）は、初回読み込み時に自動的に折りたたまれます。必要に応じてファイルヘッダーをクリックして展開してください。

## 検索

`Cmd/Ctrl+F` で検索バーを開きます。表示中のすべてのdiffコンテンツを横断して検索できます。
