# CLAUDE.md — Tasuki 開発ガイド

レビュー補助ツール: Claude Code と人間が襷を渡し合いながらゴールに向かう

## プロジェクト構成

- **フロントエンド**: React 19 + TypeScript + Vite 7 (port 1420)
- **デスクトップ**: Tauri v2 (`src-tauri/`)
- **状態管理**: Zustand 5 (`src/store/`)
- **Diff描画**: `@pierre/diffs` (Shadow DOM内でレンダリング)
- **Markdown**: react-markdown + remark-gfm + rehype
- **Mermaid**: beautiful-mermaid
- **シンタックスハイライト**: Shiki
- **ターミナル**: xterm.js 6 + portable-pty (Rust)
- **テスト**: Vitest (unit) + Playwright (e2e)

## コマンド

| 用途 | コマンド |
|------|---------|
| ユニットテスト | `npm test` |
| E2E テスト | `npm run test:e2e` |
| Lint | `npm run lint` |
| ビルド | `npm run build` |
| 開発サーバー | `npm run dev` |

## 実装後の自動チェック (CI / Claude Code で実行可能)

変更を push する前に必ず以下を通すこと:

1. `npm run lint` — ESLint
2. `npm test` — Vitest ユニットテスト
3. `npm run build` — TypeScript コンパイル + Vite ビルド

## 実装後の手動確認チェックリスト

以下は自動テストではカバーしきれない項目。`npm run dev` でアプリを起動し、実装内容に応じて該当する項目を確認すること。

### 1. diff表示 (対応E2E: ux1)

- [ ] アプリ起動後、左サイドバーの「Changed Files」にファイル一覧が表示される
- [ ] サイドバーのファイル名をクリックすると、そのファイルのdiffまでスクロールする
- [ ] ツールバーの「Split」ボタンを押すとside-by-side表示、「Unified」ボタンを押すと上下結合表示に切り替わる
- [ ] ツールバーの「Scroll / Wrap」ボタンで長い行の折り返し/横スクロールが切り替わる
- [ ] ツールバーの「Expand / Collapse」ボタンで未変更行の展開/折りたたみが切り替わる
- [ ] ファイルヘッダー(ファイル名部分)をクリックすると、そのファイルのdiffが折りたたまれる
- [ ] 追加行が緑、削除行が赤、変更ファイル/新規ファイル/削除ファイル/リネームのアイコンが正しく表示される

### 2. 行コメント (対応E2E: ux2)

- [ ] diff内の行番号付近にマウスホバーすると「+」ボタンが表示される
- [ ] 「+」ボタンをクリックすると、その行の下にコメント入力フォーム(textarea)が開く
- [ ] 複数行を選択(ドラッグ)してから「+」ボタンを押すと、選択範囲がコメント対象になる(フォームに `L10-L15` のように範囲が表示される)
- [ ] テキストを入力して「Add Comment」ボタンまたは Cmd/Ctrl+Enter で送信すると、diff内にコメントが表示される
- [ ] 「Cancel」ボタンまたは Escape キーでフォームが閉じる
- [ ] 送信後、画面下部の Review Panel にもそのコメントが `ファイル名:L行番号` 付きで表示される

### 3. コメントのコピー (対応E2E: ux3)

- [ ] Review Panel 内のコメント横のコピーボタン(クリップボードアイコン)をクリックする
- [ ] クリップボードにコピーされた内容にファイルパス、行番号(`ファイル名:L12`)、コードスニペット、コメント本文が含まれる
- [ ] テキストエディタに貼り付けて、フォーマットが `ファイル名:L行番号 > コードスニペット コメント本文` の形式になっている

### 4. 全コメント一括コピー (対応E2E: ux4)

- [ ] コメントが0件のとき、Review Panel の「Copy All」ボタンが disabled(押せない状態)になっている
- [ ] コメントを1件以上追加すると「Copy All」ボタンが enabled になる
- [ ] 「Copy All」をクリックし、クリップボードの内容に `## Review Result:` ヘッダーと全コメントが含まれる

### 5. ドキュメント並列表示 (対応E2E: ux5)

- [ ] ツールバーの「Split」タブがデフォルトで選択状態になっている
- [ ] 画面が左右に分割され、左にdiff、右にMarkdownドキュメントが表示される
- [ ] サイドバーの「Documents」セクションからドキュメント名(例: `architecture.md`)をクリックすると、右ペインにその内容が表示される
- [ ] Markdownの見出し・リスト・コードブロックが正しくレンダリングされる
- [ ] Mermaidの図がある場合、SVGとして描画される(architecture.mdにサンプルあり)
- [ ] Mermaidの図にマウスホバーすると拡大ボタン(↔アイコン)が表示される
- [ ] 拡大ボタンをクリックするとフルスクリーンのズームモーダルが開く
- [ ] ズームモーダル内で +/- ボタンやマウスホイールで拡大縮小できる
- [ ] ズームモーダル内でドラッグでパン(移動)できる
- [ ] パーセンテージ表示をクリックするとズームがリセットされる
- [ ] Escapeキーまたは背景クリックでモーダルが閉じる
- [ ] 左右の境界線をドラッグして、ペインの幅を変更できる
- [ ] 大画面でも右ペインの幅が最大900pxに制限される（diffが主役になる）

### 5b. Viewerモード (対応E2E: ux6)

- [ ] ツールバーの「Viewer」タブをクリックすると3ペインレイアウト（サイドバー | Markdown | ターミナル）に切り替わる
- [ ] Viewerモードではdiff関連のツールバーコントロール（Split/Unified, Scroll/Wrap, Expand/Collapse, 統計）が非表示になる
- [ ] サイドバーの「Changed Files」セクションが非表示になる
- [ ] サイドバーの「Documents」「Design Docs」セクションは表示される
- [ ] サイドバーの「Add Folder」ボタンが表示される
- [ ] ドキュメントをクリックすると左ペイン（メイン領域）にMarkdownが表示される
- [ ] 右ペイン（ターミナル）の幅が最大900pxに制限される（Markdownが主役になる）
- [ ] 左右の境界線をドラッグしてペインの幅を変更できる

### 5c. 外部フォルダ閲覧 (Tauri環境のみ)

- [ ] 「Add Folder」ボタンをクリックするとフォルダ選択ダイアログが開く
- [ ] フォルダを選択すると、サイドバーにそのフォルダ名のセクションが追加される
- [ ] 追加されたフォルダ内の `.md` ファイルがツリー表示される
- [ ] 外部フォルダのMarkdownファイルをクリックするとその内容がプレビューされる
- [ ] フォルダセクションの「✕」ボタンでフォルダを除去できる
- [ ] 複数のフォルダを追加でき、それぞれ独立したセクションになる

### 6. コメントの解決/Approve/Reject

- [ ] Review Panel 内のコメント横のチェックマーク(✓)ボタンを押すと、解決メモ入力欄が出る
- [ ] メモを入力(または空のまま)して「確定」を押すと、コメントが「解決済み」になる(取り消し線・薄い表示)
- [ ] 解決済みコメントの「↩」ボタンで解決を取り消せる
- [ ] コメントの「✕」ボタンでコメントを削除できる
- [ ] 未解決コメントがある間は「Approve」ボタンが disabled
- [ ] 全コメントを解決すると「Approve」ボタンが押せるようになる
- [ ] 「Reject」ボタンは未解決コメントがあっても押せる

### 7. その他のUI操作

- [ ] ツールバーの「Terminal」タブでターミナル画面に切り替わる(Tauri環境のみ)
- [ ] ターミナル画面では右側にドキュメントが並列表示される
- [ ] Cmd/Ctrl+F でdiff内検索バーが開く
- [ ] サイドバーの「Design Docs」セクションがあれば、設計ドキュメントを表示できる

### 8. Tauri環境固有 (npm run tauri dev)

- [ ] コピーボタンでクリップボードにコピーできる(Tauri clipboard API経由)
- [ ] ファイル変更を検知してdiffが自動更新される(ファイルウォッチャー)
- [ ] レビューセッションがアプリ再起動後も復元される(.tasuki/reviews/に保存)

## 注意事項

- Vitest は `environment: "node"` で動作する。ブラウザ API (`window`, `document` 等) を参照するコードにはガード (`typeof window !== "undefined"`) を入れること
- `src/store/index.ts` の `__zustandStore` 公開は DEV モード + ブラウザ環境のみ
- diff描画は `@pierre/diffs` が Shadow DOM 内で行う。Playwright からは CSS セレクタで pierce 可能だが、hover イベントは Shadow DOM に阻まれるため、E2E テストでは `openCommentForm()` でストア経由で操作する
- E2E テストは `localhost:1420` の開発サーバーに接続する。`reuseExistingServer: true` なので事前に `npm run dev` を起動しておくと速い
- Tauri 環境外（`npm run dev` のブラウザ開発時）では `tauri-api.ts` がモックデータを返す。新しいコマンドを追加した場合は `mockInvoke` にもケースを追加すること
- レビューセッションは `.tasuki/reviews/` に永続化される。コミットゲートは `/tmp/tasuki/{repo}/{branch}/review.json` に書き込まれる
- 設計書は `~/.claude/designs/{repo-name}/` から読み取られる。パストラバーサル防止のバリデーションあり
